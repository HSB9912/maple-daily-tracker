• 캐릭터명 옆 ▶ 클릭 → 정산 이력 펼치기/접기
• 합산 메소가 마이너스면 빨간색 표시 (선금이 누적보다 많음 = 부주가 줘야 할 빚)
• 합산 메소가 0이 되면 자동으로 "정산 완료" 섹션으로 이동 (전체 정산과 동일)
• 각 이력 항목별로 ↩되돌리기 가능 (기존 전체 롤백 대신)

📝 구현 요약

  • 데이터 구조 확장:
    settlements[ch].entries = [
      { id, date, type: 'cash'|'meso'|'advance', amount, mesoEquiv, memo },
      ...
    ]
  • 잔여 누적 메소 = (전체 누적 메소) − (entries의 mesoEquiv 합)
  • 전체 정산 = 현재 잔여만큼 'cash' 또는 'meso' entry 1개 추가
  • 부분정산 = 사용자 입력 금액만큼 entry 추가 (여러 번 가능)
  • 선금 = 잔여가 음수가 되어도 entry 추가 가능 (마이너스 누적 허용)
  • 각 entry는 개별 되돌리기 가능 (entries.splice)
  • 마지막 정산일(lastDate) 개념 유지 — 이전 잔여 = 0이 됐던 가장 최근 시점
  • 기존 history[] 구조와 호환되도록 마이그레이션
