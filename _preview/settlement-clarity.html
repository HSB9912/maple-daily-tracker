정산 명확화 — 3가지 시안

변동가 재화(다조·이슬 등 경매장 시세) 때문에 잔여 메소가 출렁이고, 부분정산 후에도 부주정산 표가 파란 누적 그대로 보여서 "이거 정산해야 되나?" → "아니네" 가 반복되는 문제를 해결하는 시안들.

🐛 현재 동작이 만드는 혼란 — 코드 기준

  1. grossMeso = Σ(qty × 현재 단가) → 단가 바꾸면 전체가 즉시 재계산
  2. paidMeso는 entry에 메소 값으로 박혀있어서 안 변함 → 잔여 = gross − paid 가 한쪽만 흔들림 → 잔여가 갑자기 마이너스로 갈 수 있음
  3. 부분정산(현금/메소/선금)은 lastDate를 건드리지 않아 → 일별 입력 그리드와 부주정산 표의 재화 수량은 처음 누적부터 그대로 파란색으로 표시
  4. 그래서 표만 슥 보면 "어 다 그대로 쌓여있네, 정산해야겠다" → 펼쳐보고 "아 이미 100M 받았네" 의 반복
시안 1

정산 스냅샷 락 — 정산 시점의 시세를 entry에 고정

축: 데이터 모델

정산 버튼 누르는 순간 그 시점의 재화 단가 전체와 당시 grossMeso를 entry에 저장. 이후 단가를 아무리 바꿔도 이 정산만큼은 영향 없음. 새로 쌓이는 부주만 새 단가로 계산.

🔒
캐릭터A · 잔여 122,600,000 메소 = 현재 누적 222,600,000 − 정산된 100,000,000 (스냅샷 잠금)
📋 정산 이력 — 각 entry는 당시 시세를 박제
2026-05-25 현금
1,500원 받음 → 100,000,000 메소 차감 스냅샷: 다조 6,800,000, 우유 800,000, 1억=1,500원
2026-05-22 메소
50,000,000 메소 받음 스냅샷: (메소만 받았으므로 단가 영향 없음)
⚠
다조 시세가 6,800,000 → 7,200,000 변경됨 — 기록된 정산은 영향 없음, 현재 미정산 분만 재계산됨 (+6,000,000 메소)

이 시안의 효과

상황현재시안 1
정산 후 다조 시세 변동잔여 메소가 같이 흔들림 (이미 받은 분도)받은 분은 고정, 미정산 분만 흔들림
이력에 "그때 얼마짜리였지?" 추적cashRateAtTime만 있음전 단가 스냅샷 + 모달에서 표시
마이너스 잔여 발생시세 변동만으로도 마이너스 가능선금 모드일 때만 발생 (정상 동작)
시안 2

정산 후 시각 마킹 — 이미 받은 만큼은 회색·취소선

축: 시각 디자인

부분정산 entry가 추가되면, 그 entry에 해당하는 가장 오래된 일별 기록부터 "처리됨"으로 마킹. 파란 글씨 → 회색·취소선으로 바뀌어서 표만 봐도 "여기까지는 이미 줬다"가 한눈에 보임.

• 일별 입력 그리드(메인 화면)에서도 정산 처리된 셀은 회색·취소선 → 지금 "정 산 완 료" 오버레이를 부분정산 단위로 확장
• 변동가 재화는 정산 시점 단가로 일별 메소값을 박제(시안 1과 결합 시)

시안 3

잔여 중심 뷰 — 표는 "줘야 할 것"만 보여줌

축: 상호작용 / 정보 구조

부주정산 표의 재화 수량을 "미정산분만"으로 기본 표시. 이미 정산된 만큼은 작은 회색 취소선으로 위에 살짝 보여줘서 "총 누적 - 받은 분 = 줄 분"이 셀 하나에서 다 보임. 상단 토글로 "전체 누적" / "잔여만" 전환.

🎯 추천 조합 — 시안 1 + 시안 3

시안 1(스냅샷 락)으로 데이터 무결성을 잡고, 시안 3(잔여 중심 뷰)로 표의 혼란을 동시에 잡는 게 가장 효율적. 시안 2는 일별 그리드까지 손대야 해서 작업량이 큼.

시안 해결하는 것 손대는 범위 위험
1 · 스냅샷 락 변동성 → 잔여 출렁임 entry 구조 확장 + 마이그레이션, 모달 표시 추가 낮음 — 기존 entry는 cashRateAtTime만 있어도 fallback 가능
2 · 일별 마킹 시각 혼란 (메인 그리드까지) renderSheet + getAccumulated 전반 손봐야 함 높음 — "어떻게 차감을 일자별로 안분?" 정의 필요
3 · 잔여 중심 뷰 시각 혼란 (부주정산 표만) renderHistory의 셀 렌더링 + 토글 추가 중간 — 재화별 안분 방식 정해야 함 (메소비중 안분 추천)
진행 순서 제안: ① 시안 1만 먼저 적용 → 데이터부터 정리 → ② 잔여 뷰(시안 3) 추가 → ③ 쓰다가 일별 그리드까지 마킹 원하면 시안 2.
